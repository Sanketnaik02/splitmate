1. PROJECT OVERVIEW
====================

SplitMate is a local-first expense splitting application that
helps groups of people track shared expenses and settle debts
easily. It works entirely offline using your browser's Local
Storage — no accounts, no server, no internet required.

The app is built with:
  • React (UI framework)
  • Vite (build tool)
  • Tailwind CSS (styling)
  • Local Storage (data persistence)

4. USER FLOW
=============

1. OPEN the app → Sign In page appears
2. SIGN UP with name, email, password → auto logged in
3. DASHBOARD shows your overall balance and groups
4. CREATE A GROUP → "Goa Trip" → auto-added as admin
5. ADD MEMBERS → type names (Sanket, Rahul, Amit)
6. ADD EXPENSE → ₹3000 for Hotel → split equally
7. VIEW BALANCES → see Rahul owes ₹1000, Amit owes ₹1000
8. SETTLE UP → record payment when Rahul pays Sanket
9. VIEW SETTLEMENT HISTORY → see all past settlements

All data persists automatically in your browser. No save button
needed. Refresh the page — everything stays.

5. DASHBOARD EXPLANATION
=========================

The Dashboard is the home screen. It shows:

  ┌─────────────────────────────────────┐
  │  OVERALL BALANCE                     │
  │  ─────────────────────────────────   │
  │  💰 Total Expenses    ₹10,000        │
  │  💳 You Paid          ₹6,000         │
  │  ─────────────────────────────────   │
  │  📈 You Receive       +₹4,000        │
  │     Others owe you                   │
  │  📉 You Owe           -₹2,000        │
  │     To other members                 │
  │  ─────────────────────────────────   │
  │  Net Balance          +₹2,000        │
  │  You are owed ₹2,000 overall         │
  └─────────────────────────────────────┘

  • Total Expenses: Sum of ALL expenses across all groups.
  • You Paid: Sum of all expenses where you were the payer.
  • You Receive: Total amount others owe you (calculated from
    each group's balance).
  • You Owe: Total amount you owe others (calculated from
    each group's balance).
  • Net Balance: (You Receive) - (You Owe). Positive means
    others owe you more than you owe them.

Below the balance card:
  • Your Groups — horizontal scrolling list of all groups.
  • Recent Activity — last 10 expenses and settlements.

At the bottom:
  • Quick Action (+) button — add expense, create group,
    or settle up.

6. EXPENSE CALCULATION LOGIC
==============================

When you add an expense, SplitMate stores:
  • Amount (e.g. ₹3000)
  • Who paid (e.g. Sanket)
  • Split type and details

EQUAL SPLIT EXAMPLE:
  Expense: Hotel = ₹3000
  Paid by: Sanket
  Split among: Sanket, Rahul, Amit (3 people)
  Each share: ₹3000 ÷ 3 = ₹1000

  Effect:
  • Sanket paid ₹3000, but owes ₹1000 (his share)
  → Sanket is owed ₹2000 total
  • Rahul owes ₹1000 (his share)
  • Amit owes ₹1000 (his share)

BALANCE COMPUTATION:
  For each person in a group:
    1. Start with balance = 0
    2. Add the amount of each expense they paid
    3. Subtract their share from each expense
    4. Add completed settlements they sent
    5. Subtract completed settlements they received

  Balance > 0 → person is owed money
  Balance < 0 → person owes money
  Balance = 0 → all settled up

============================================================
7. SETTLEMENT LOGIC
=====================

When you settle up:

  1. The app shows suggested payments based on current balances
     (simplified to minimize number of transactions).
  2. You tap "Pay" or "Record" to record a settlement.
  3. The settlement is stored as a record with:
     • From: who paid
     • To: who received
     • Amount
     • Status: "completed"
  4. Balances are recalculated considering settlements.
  5. The group shows updated balances.

  DEBT SIMPLIFICATION:
  Instead of everyone paying everyone else, SplitMate finds the
  minimal set of payments to settle all debts.

  Example:
    • Sanket is owed ₹1000 by Rahul and ₹1000 by Amit
    Instead of two payments, the app may suggest:
      Rahul → Sanket ₹1000
      Amit → Sanket ₹1000
    (This is already optimal in this case)

8. LOCAL STORAGE STRUCTURE
============================

All data is stored in your browser's Local Storage with the
prefix "splitmate_". Here are the keys and their contents:

  Key: splitmate_users
  Value: Array of user objects
    { id, displayName, email, password, photoURL, phone,
      defaultCurrency, createdAt }

  Key: splitmate_groups
  Value: Array of group objects
    { id, name, category, description, createdBy, currency,
      inviteCode, createdAt }

  Key: splitmate_members
  Value: Array of member objects
    { id, groupId, userId, displayName, role, createdAt }
    (role is "admin" for creator, "member" for others)

  Key: splitmate_expenses
  Value: Array of expense objects
    { id, groupId, description, amount, paidBy, date, category,
      splitType, splitDetails, createdAt }
    splitDetails is an object: { userId1: share1, userId2: share2 }

  Key: splitmate_settlements
  Value: Array of settlement objects
    { id, groupId, fromUserId, toUserId, amount, note, status,
      settledAt, createdAt }
    (status is "completed" after settlement is recorded)

  Key: splitmate_session
  Value: Single user ID string (current logged-in user)

  Key: splitmate_theme
  Value: "light" or "dark"

  Key: splitmate_<collection>
  Value: Always an array of objects with unique "id" fields.

  Note: All IDs are auto-generated strings. All objects have
  a "createdAt" timestamp.

10. FUTURE FIREBASE PLAN
=========================

When moving to Firebase (cloud):

  1. Create a Firebase project at https://console.firebase.com
  2. Enable Authentication (email/password + Google)
  3. Create Firestore database
  4. Replace storage.js with Firebase SDK calls
  5. Add real-time listeners (onSnapshot)
  6. Add image upload for avatars (Firebase Storage)
  7. Deploy to Firebase Hosting

Key changes:
  • store.add()    → firebase.firestore().collection().add()
  • store.get()    → firebase.firestore().doc().get()
  • store.update() → firebase.firestore().doc().update()
  • store.remove() → firebase.firestore().doc().delete()
  • store.where()  → firebase.firestore().collection().where()

Schema maps directly — same field names, just stored in
Firestore collections instead of Local Storage.

14. DEPLOYMENT GUIDE (Vercel)
===============================

Prerequisites:
  • Node.js installed
  • Git installed
  • A Vercel account (free at https://vercel.com)

Steps:
  1. Open terminal in the SplitMate folder
  2. Run: npm install
  3. Run: npm run build
     (Verify build succeeds with no errors)
  4. Install Vercel CLI: npm install -g vercel
  5. Run: vercel login
     (Follow browser prompt to authenticate)
  6. Run: vercel
     (Select: Yes to build, Yes to deploy)
  7. Vercel will provide a URL like:
     https://splitmate-xxx.vercel.app
  8. Open the URL on your phone to test

Alternative (without CLI):
  1. Push your code to a GitHub repository
  2. Go to https://vercel.com/new
  3. Import your GitHub repository
  4. Vercel auto-detects Vite framework
  5. Click "Deploy"
  6. Your app is live at a .vercel.app URL

The vercel.json file in the project root ensures:
  • SPA routing works (all routes redirect to index.html)
  • Build command is set to "npm run build"
  • Output directory is "dist"
